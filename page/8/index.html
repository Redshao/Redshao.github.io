div里面有个img标签，那么img会给它的父容器增加3px的高度

发表于 2016-05-10   |   分类于 前端   |  

今天有个同学问我这样一个问题，div里面有个img标签，给img一个图片，div没有设置高度，那么按照正常的应该是这个div等于img的高度，但是浏览器查看了发现div莫名的多了3px，苦思冥想很久一直没有答案，晚上墙外看了下，在http://stackoverflow.com/里面找到了答案

原问题：

When I change my website to Every img element that is wrapped inside a DIV has a 3px bottom margin to it even though that margin is not defined in CSS. In other words, there are no style attributes that are causing that 3px bottom margin.

Now let’s say haha.jpg is 50x50, and .placeholder is set to display: table. Strangely the height dimensions of .placeholder in my observation is 50x53…

Has anyone encountered this anomaly before and fixed it?

答案：

This problem is caused by the image behaving like a character of text (and so leaving a space below it where the hanging part of a “y” or “g” would go), and is solved by using the vertical-align CSS property to indicate that no such space is needed. Almost any value of vertical-align will do; I’m fond of middle, personally.

意思就是img标签会自动加上3px的空白字符，可以通过vertical-align：middle或display:block或display:inline-block干掉它

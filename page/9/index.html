css中的dispaly:inline-block键值对发现

发表于 2016-05-04   |   分类于 前端   |  

css元素的显示方式有三种：

行内元素（display:inline）

1）一行可以显示多个行内元素（优点）
2）行内元素的宽高由其内容来决定
3）不能设置宽高

块级元素（display:block）

1）一行只能有一个块级元素
2）块级元素默认宽度占浏览器屏幕的一整行
3）可以设置宽高（优点）

行内块级元素（display:inline-block）

1）集合了行内元素和块级元素的优点

My first project

发表于 2016-01-20   |   分类于 产品   |  

我是一个记得快也忘得快的人，所以记录下自己做过的第一个项目，也算是对如今成长的自己的一个交代

第一个项目是我来公司大概大概三个月的时候所做的，之前我是一个销售，对于如何做产品只有一个模糊的认知，没有参与过也没有了解过，所以刚接触项目助理这个行业我就从同事那里了解下我的本职需要会哪些必备的技能，同事对我说；“Axure”，我百度了下Axure啥玩意，渐渐了解到是一款原型制作软件，好吧，吃饭的家伙必须得熟，所以刚来的那段时间基本上都是在了解Axure的使用，这里非常感谢人人都是产品经理这个网站，对于我这个小白来说还是有满满的收货

然后了解的越多其实越乱，正好在5月份的时候我的项目经理接到了一个项目，当天下午我和他（称他为Java吧）去客户公司了解需求，了解的大概三个小时左右，主要就是基于微信公众号开发微店，包括分销，客户自己还做了一个前台的原型demo，当时微信分销蛮火的，客户自然不想落下，不过我当时心想有Java在我打打下手就行了，，从客户那离开的时候Java顺便把客户的原型demo也要来了作为敞开，然而这对于我来说只是噩梦的开始

当天下午，Java让我按照需求开始制作原型demo，他负责需求文档，因为我没有写过需求文档，这个项目比较重要，所以他来负责关键的部分，回到公司我打开Axure停留了很久，然后还是电话给Java说我第一次做，还是你来做原型吧，我打打下手，Java想了下还是决定让我做，说这个项目和我们有些产品有共同点，你直接截图就行，我心想那就截图吧，花了一个下午的时间把相似产品后台的界面截图和客户给的前台原型demo拼凑成一个原型，然后当晚邮件发给Java，Java带着原型和需求文档去和客户进行了第二次沟通，客户对我们公司实力和Java还是比较信任的，所以只是大致看了下原型和文档就定下来了开发的周期安排

接下来原型自然落到公司负责的技术总监那，技术总监只有一句话：“这是什么玩意？”，前台后台功能对不上，没有细节，没有细节，没有细节，当晚Java给我说过之后我就知道事情大了，因为这些不是没有考虑过，而是我个人的意识一直以为页面的分页在原型中没有技术也会做，因为这是很正常的一件事，但是技术都是按照原型给的去coding，不会添加理所当然的东西，苦逼的修改开始了，和Java每天修改到晚上11甚至更久，因为不能在我们手上浪费太多的时间，所以召集设计师开始设计交互，这段时间的成长一天相当于一个月，毫不夸张的说，我天天泡在技术部和设计部，一有疑问立马进行解答，让项目保持通顺

回家时我也时常反省我错的有多离谱：

1不注重细节，不能说不注重吧，应该说没有细节，这是对我的一次沉重的打击

2沟通很重要，不管对于Java来说，设计师、前端、程序猿或者测试来说，良好的沟通很重要，很重要，你说的表达你所想的，而他能从你说的理解你所想的

3Axure就是一个工具，不用害怕，需求文档亦然，只是一个文字表达罢了，逻辑清楚，描述完整就是最好的

4要有审美能力，有人不爱钱没人不爱美，你我都是如此

5懂得自我调节，因为你面对的大多数是汉子，火药有火就会炸

谨记！！！

PHP的HTTP_USER_AGENT

发表于 2015-12-07   |   分类于 PHP   |  

1用$_SERVER[‘HTTP_USER_AGENT’]系统函数获取当前用户的浏览器标识

2用strpos函数来判断$_SERVER[‘HTTP_USER_AGENT’]中是否有自己需要的浏览器标识，比如猎豹浏览器的LBBROWSER，其他浏览器都可以判断

3用if语句来输出自己想要的字符串或者图标（个性化）

备注：我之前看过一个博客的用户评论，每个用户的评论会有个头像和自己评论使用的浏览器图标，比如火狐、chrome等等，应该用这种函数可以完美实现
